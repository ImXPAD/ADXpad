Introduction

This is an EPICS areaDetector driver for XPad detectors from ImXPad.

This EPICS interface is made via the tcp/ip server provided with the camera. For this driver to work the server (pci or usb) must be running on the same IP and Port as stated in the startup script st.cmd or in the configuration function.

Building


To install this module please follow the instructions relative to any areaDetector Driver available here .
If you have limited memory or have to share the memory between a lot of processes you might want to change the values of MAX_RETURN_SIZE to an inferior value and XPAD_SUPP_DELAY to a greater value.

You can edit all the defines at the begining as well to adapt everything to your own system !

WARNING: if every function fails your XPAD_SUPP_DELAY is probably too short .



Configuration

Before you start, edit the st.cmd file replacing both XpadXXXServer and XpadAbort Ip adresses by the real one

You can also edit the NELEMENTS value of the image1 record, it is by default set to a value allowing 10 modules detectors (s700, wos...) you can replace it with a smaller value if you have a smaller detector!


The xpad driver is created with the xpadConfig command, either from C/C++ or from the EPICS IOC shell. You can see it in the st.cmd file

int xpadConfig(const char *portName, const char * serverPort,int maxBuffers, size_t maxMemory, int priority, int stackSize)

Most of these parameters are the standard ADDriver parameters . It might be wise to use the startup script for everything including the Abort to work

Restrictions

The following are some current restrictions of the areaDetector xpad driver:

-No Continuous mode 

-Not a wide choice of file formats

-MEDM & CS-Studio interfaces only


Known bugs & Problems

  1. Random bug:  Driver is not capable of sending or reading anything to server, often caused by asking too many things at once

                    Solution: - One solution is to open the prompt and clic on read a few times to see what is stuck in the channel and empty it

                                    -Reboot driver

     2. Abort when the detector is between 2 exposure might lock the program(Server bug)

                                    -Reboot Detector (&Driver)

                                    -Abort twice or more and wait until the server timesout(should be fixed in the next XpadServer version)

    3. Calibration files are corrupted 

                                    -Reset MAX_RETURN_SIZE in xpad.h and rebuild. We recommend 4096bytes but you can use a smaller value depending on the detecor model

 

    4.Everything fail


                                    -Make sure XPAD_SUPP_DELAY is not too small

                                    -Plug in the detector

                                    -Tell us

 
    5. Not really a bug but it is not obvious when a function failed

                                    -Keep an eye on the iocsh logs  & server
